Methodologies

In this section, we will clarify the methodology used to perform a cluster analysis on indicators data. The analysis can be done on two types of data: multi-year average of monthly data for the years 1983 – 2016, and monthly data for each year from 1983 to 2016. In the Basic mode, the analysis is done only on the cross-year average of monthly data. The user is not allowed to select a custom range for the values of the selected indicators. The clustering algorithm, which in this case is the Hierarchical Agglomerative Clustering (HAC), is thus applied on the whole selected indicators data. In the Advanced mode, the analysis can be done on the two aforementioned types: the multi-year average of monthly data for the years 1983 – 2016, and the monthly data for each year from 1983 to 2016. In this mode, the user can select custom ranges for each selected indicator data. The clustering algorithms that can be used include: Density-Based Spatial Clustering of Applications with Noise (DBSCAN), Hierarchical Density-Based Spatial Clustering of Applications with Noise (HDBSCAN) and Hierarchical Agglomerative Clustering (HAC).

1. DBSCAN - Density-Based Spatial Clustering of Applications with Noise: This technique is used to find arbitrary shaped clusters and to detect outliers (i.e. clusters with noise). DBSCAN requires two key parameters:

  • EPS: Specifies how close points should be to each other to be considered neighbors and a part of a cluster. If the distance between two points is lower or equal to eps, these two points are considered neighbors.

  • minPts: This parameter specifies the minimum number of data points to form a dense region and, thus, a cluster.

Points that are in low-density regions are marked as outliers.




2. HDBSCAN - Hierarchical Density-Based Spatial Clustering of Applications with Noise : This algorithm extends DBSCAN by converting it into a hierarchical clustering algorithm. The parameter that should be defined for our implementation is Minimum Cluster Size, which specifies the minimum number of points that a cluster should have. By default, this parameter is set to 10.



3. HAC - Hierarchical Agglomerative Clustering :HAC uses a bottom-up approach to form clusters. It starts by considering each point as a cluster, and merges pairs of clusters as moving up the hierarchy. The algorithm keeps merging until reaching the optimal number of clusters. We rely on the Silhouette method to define that optimal number of clusters from a user-entered range. This range is set by default to [2 – 5].
